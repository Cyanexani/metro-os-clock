● I dug into the code. Here's the honest diagnosis for each thing you hit — and
  note I just reverted the rework a moment ago, so a few of these ("no world
  clock text", "no alarms & clock text") are side effects of that revert, while
  others (flicker, zoom overlap) are real bugs that exist in both versions.

  Root causes

  1. Flicker when seconds are shown — real bug,
  components/compound/AnimatedTime.js:20. Every time the value changes it plays
  a fade-out→fade-in ("cross-dissolve"). With seconds off it fires once a minute
  (invisible); with seconds on it fires every second → that's the flicker. Real
  WP just swaps the digits, no fade.

  2. Why the WP map was removed / this one is here — it was never a real WP map.
  assets/world-map.png has been the same generic equirectangular map (with the
  Antarctica blob) since the very first map commit c171093. There is no 
  WP-authentic map in git history to restore — I'd have to source or generate
  one.

  3. Zoomed map glitches + overlaps the timer when you swipe horizontally — real
  bug. The map zooms via transform: scale(2.8) (useMapZoom.js:12). The outer
  mapContainer in WorldClock.js:264 has no overflow: 'hidden', and on Android a
  scaled child isn't clipped by the parent unless the parent explicitly clips.
  So when you swipe between tabs while zoomed, the enlarged map bleeds sideways
  over the stopwatch/timer screens. It also never resets zoom when you leave the
  tab.

  4. No "WORLD CLOCK" overline on the map — that text lived in WorldClockHeader,
  which my revert just deleted. The reverted (old) world clock never had that
  overline.
