### (unknown source)

1. Windows Phone 8 defines exactly four Segoe WP font family theme resources: PhoneFontFamilyNormal = 'Segoe WP', PhoneFontFamilyLight = 'Segoe WP Light', PhoneFontFamilySemiLight = 'Segoe WP Semilight', and PhoneFontFamilySemiBold = 'Segoe WP Semibold'.
2. The exact Windows Phone theme font-size scale (in XAML logical pixels) is: Small 18.667, Normal 20, Medium 22.667, MediumLarge 25.333, Large 32, ExtraLarge 42.667, ExtraExtraLarge 72, and Huge 186.667 — the last being the size used for the giant light-weight numerals (PhoneTextHugeStyle uses PhoneFontFamilyLight at PhoneFontSizeHuge).
3. Page/pivot title typography is defined by PhoneTextTitle1Style, which uses the Semilight font family at 72px (PhoneFontSizeExtraExtraLarge); group headers (PhoneTextGroupHeaderStyle) use Semilight at 32px with the subtle (grey) foreground brush — giving exact weight/size targets for replicating pivot headers and section headers.
4. Windows Phone's standard layout metric is 12px: the standard margin (PhoneMargin) is 12, horizontal margin is '12, 0', touch-target overhang is 12 (large overhang '12, 20'), and standard border/stroke thickness is 3 — key spacing constants for a faithful Metro layout grid.
5. The application bar and status bar share a dedicated 'chrome' color distinct from the page background: PhoneChromeBrush/PhoneChromeColor is defined as 'the color of the status bar and application bar', and accent highlighting uses PhoneAccentBrush ('foreground color to single-out items of interest') whose actual value depends on the user's chosen accent theme rather than a fixed hex in this spec.

### (unknown source)

6. The Windows Phone app bar is fixed at 72 pixels (height in portrait, width in landscape) and this size cannot be modified.
7. App bar menu item text is automatically converted to all lowercase characters by the OS to maintain consistency with the rest of the device.
8. Tapping the app bar ellipsis reveals icon button labels and slides up a list of text menu items from underneath the icon buttons, with built-in show/hide animation.
9. Microsoft's guidance is to use default system theme colors for the app bar (not custom colors), and app bar opacity should only be 0.0, 0.5, or 1.0 — at 1.0 the page is resized to exclude the bar, while lower values overlay the page content.
10. App bar menus should have at most five menu items (to avoid scrolling) with menu text of 14–20 characters maximum, and in landscape the bar moves to the side of the screen and always uses the default size.

76. The original Windows Phone tilt-press effect uses a maximum tilt angle of 0.3 radians (about 17.2 degrees) and a maximum press depression of 25 pixels along the Z axis.
77. The tilt release animation waits 200 ms after the finger lifts before animating, then returns rotation and depression to zero over 100 ms.
78.
